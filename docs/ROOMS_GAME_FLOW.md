# Комнаты и игровой цикл

## 1. Создание и поиск комнаты

- Админ создаёт шаблон комнаты (`room_template`).
- Клиент вызывает `POST /api/room/find` с игровыми параметрами.
- Backend:
  - ищет активную `WAITING` комнату с подходящими параметрами;
  - проверяет свободные места и остаток времени (>5 сек);
  - если подходящей нет — создаёт новую комнату.

Важно: `find` не присоединяет игрока к комнате.

## 2. Присоединение к комнате

- Вход только по `roomId` или `shortId`.
- Пользователь передаёт:
  - либо конкретные места (`seatNumbers`),
  - либо количество случайных мест (`seatsCount`).
- Ограничение: пользователь может занять максимум 50% мест комнаты.
- Если место занято — ошибка.
- При входе резервируется стоимость: `entryCost * число_мест`.

## 3. Таймер раунда

- Таймер стартует после входа первого игрока (`firstPlayerJoinedAt`).
- По истечении:
  - если мест не хватает — добор ботами;
  - затем завершение комнаты и розыгрыш.

## 4. API запуска раунда и определения победителя

Боевой запуск раунда:

- `POST /api/rooms/{roomId}/finish`

Что делает:

1. Проверяет состояние комнаты.
2. Финализирует резервы (`commit`).
3. Считает веса игроков.
4. Запрашивает рандом в `random-service`.
5. Определяет победителя.
6. Начисляет приз.
7. Пишет `game_results`, `game_result_players`, `round_event_logs`.

Сервисный endpoint:

- `POST /api/game/winner` — вычисление победителя для переданного набора игроков.

## 5. Формула призового фонда

Призовой фонд рассчитывается как:

`prizeFund = entryCost * maxPlayers * winnerPercent / 100`

Этот же расчет возвращается в API шаблонов и в состоянии комнаты.

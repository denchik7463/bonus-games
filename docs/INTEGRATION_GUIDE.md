# Встраивание в ЛК / игровой раздел

## Цель интеграции

Подключить модуль как внешний игровой backend, сохранив единый UX личного кабинета.

## Как встроить в существующий ЛК

1. ЛК получает токен пользователя через `POST /api/auth/login`.
2. Хранит токен и отправляет его как `Authorization: Bearer <token>`.
3. Подтягивает профиль и баланс:
   - `GET /api/profile/me`
   - `GET /api/wallet/balance`
4. Для игры:
   - `POST /api/room/find` -> найти/создать подходящую комнату
   - `POST /api/rooms/{roomId}/join` -> выбрать/купить места
   - `POST /api/rooms/{roomId}/boost/activate` -> активировать буст на место
5. Подписка на realtime:
   - `ws://.../ws/rooms?roomId=...&token=...`
6. История игрока:
   - `GET /api/game/journal/me`

## Как передается баланс бонусных баллов

- Баланс возвращается в ответах backend API (`available`, `reserved`, `total`).
- ЛК отображает его локально и обновляет после каждой финансовой операции.

## Как передавать user-контекст

Минимум для интеграции:

- `userId`
- `username`
- `role`
- токен сессии

## BFF-подход

Проект уже содержит пример BFF/proxy-слоя во `front` (`/backend-api/[...path]`).
В корпоративном ЛК можно использовать такой же подход:

- UI -> BFF -> game-backend
- единая CORS/безопасность на стороне BFF
- централизованная обработка ошибок и ретраев.
